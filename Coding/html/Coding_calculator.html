Kalkulator - Java

#coding #java

02.10.2019

Pierwszym programem, no dobra drugim bo pierwszym było "Hello World!" ;), był kalkulator. Cały kod jest dostępny na moim GitHub'ie pod tym linkiem.

Bardzo prosty programik, wykonujący cztery podstawowe działania. Wersja kodu wrzucona na GitHub'a jest trzecią lub czwartą. Wcześniejsze wersje sprawdzał mi kolega programista (Paweł pozdrawiam Cię z tego miejsca i dzięki za cierpliwość) i mówił co można jeszcze poprawić.

Ostateczna wersja różni się głównie tym, że zastosowałem w niej Interfejs z metodą wykonującą ogólne działanie na dwóch liczbach:

public interface CalcCommand {
public double evaluate(String a, double c, double d);
public static final String ADD = "+";
public static final String MINUS = "-";
public static final String MULTIPLY = "*";
public static final String DIVIDE = "/";
public static final String RESET = "C";
}

Teraz od razu komentarz - w interfejsach nie nie trzeba pisać modyfikatorów dostępu 'public', 'static', 'final' ponieważ w interfejsach wszystkie metody są publiczne, a stałe statyczne i finalne.

Brakuje na pewno obsługi błędów, bo o ile program wyłapuje, że wprowadzono zły symbol działania to zamiast poprosić o podanie prawidłowego - po prostu się wyłącza.

Druga sprawa to testy. Napisałem kilka testów jednostkowych JUnit ale czy to jest wystarczające nie jestem pewien.

class CommandTest {
private String a;
private double c = 3;
private double d = 2;

AddCommand add = new AddCommand(c,d);
MinusCommand minus = new MinusCommand(c,d);
MultiplyCommand multiply = new MultiplyCommand(c,d);
DivideCommand divide = new DivideCommand(c,d);

@Test
void evaluateAdd() {

a = CalcCommand.ADD;
assertTrue(a.equalsIgnoreCase("+"));
assertEquals(5,add.evaluate(a,c,d));
}

@Test
void evaluateMinus() {

a = CalcCommand.MINUS;
assertTrue(a.equalsIgnoreCase("-"));
assertEquals(1,minus.evaluate(a,c,d));
}
@Test
void evaluateMultiply() {

a = CalcCommand.MULTIPLY;
assertTrue(a.equalsIgnoreCase("*"));
assertEquals(6,multiply.evaluate(a,c,d));
}
@Test
void evaluateDivide() {

a = CalcCommand.DIVIDE;
assertTrue(a.equalsIgnoreCase("/"));
assertEquals(1.5,divide.evaluate(a,c,d));
}

}
